# 项目亮点

## 用户社交模块

### 双Token认证机制

- 业务场景：
  - 移动端App（骰友记）用户希望“一次登录，长久有效”（比如一个月不用重新输入密码）。
  - 如果手机丢失，或者 Token 被劫持，长效 Token 极其危险。
- 解决方案:
  - Access Token (短效)：有效期由原来的 7 天改为 15-60 分钟。用于高频 API 请求，载体为 JWT，无状态（服务端不查库，只验签），保证 API 极其高效。
  - Refresh Token (长效)：有效期 30 天。存储在服务端（Redis）和客户端。仅用于“换取新的 Access Token”。
  - 黑名单/版本号机制：当用户修改密码或点击“强制下线”时，直接在 Redis 中删除或拉黑对应的 Refresh Token。即便黑客偷走了 Access Token，最多只能作恶 15 分钟。

### 抗高并发"双buffer"发号器

- 业务场景
  - 用户 ID 需要简短好记（如：668899）以便搜索好友，而不是像 UUID 那样长的一串乱码。
  - 需要支持可能存在的高并发注册场景，要求发号器性能极高。

- 解决方案
  - 基于“号段模式” + Redis Set：数据库记录当前发到的最大号段（MaxID），服务启动时预加载一批（如 1000 个）ID 到 Redis Set 中。
  - 双 Buffer 异步缓冲：不要等到 Redis 里的 ID 用完再加载。设置一个水位线 (Watermark)。当 Redis 中剩余 ID 低于 20% 时，异步线程立即去数据库申请下一个号段并填充到 Redis。
  - 防重复与随机: 利用Redis SPOP 命令的天然随机性，保证ID不重复且无序。
